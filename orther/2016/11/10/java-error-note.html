java常见错误

1、Too small initial heap for new size specified
错误内容：Too small initial heap for new size specified
由于jdk1.8以后取消了-XX:PermSize、-XX:MaxPermSize 选项，在jdk1.7下如果不使用该选项则会出现
“Too small initial heap for new size specified”的错误，导致jvm启动不了。
解决办法：
1、jdk1.7下增加该2项配置参数。
2、更换jdk1.8，取消该2项配置参数后即可正常运行。
